# 4.3 Git Branching: Parallel Dimensions

Have you ever seen a movie about time travel or the "Multiverse" (like Spider-Man or Everything Everywhere All at Once)? 

Where one character makes a different choice, and suddenly a brand new, parallel universe splits off from the main timeline? 

**That is exactly what Git Branching is!**

## Why do we need Branches?
Imagine you have a perfectly working website. Your boss asks you to build a completely new Checkout Page. This will take you 3 weeks to build, and the code will be messy and broken while you work on it. 

If you make changes directly to the main code, your website will be broken for 3 weeks! 

Instead, you create a **Branch**.

## The Main Branch
By default, your main timeline is called `main` (or historically, `master`). This timeline MUST ALWAYS WORK. You never put broken code in `main`.

## Branching Commands

### 1. `git branch` (Creating the Parallel Universe)
* **What it means:** "Create a copy of the universe exactly as it looks right now, and give it a name."
* **How to use it:** `git branch new-checkout-page`

### 2. `git checkout` or `git switch` (Jumping Dimensions)
* **What it means:** "Transport me into that new universe so I can start working in it."
* **How to use it:** `git checkout new-checkout-page`
* *(Pro-Tip: You can create a branch and jump into it at the same time by typing: `git checkout -b new-checkout-page`)*

## What Happens Now?
Now, you are inside your safe, parallel universe. 
You can delete files, write terrible code, completely break the website, and `git commit` as many times as you want! 

If you switch back to the `main` universe (`git checkout main`), all your messy code instantly vanishes from your screen. Your website is perfectly safe. 

When you switch back to your branch (`git checkout new-checkout-page`), your messy code magically reappears!

## 🛠️ Practical Exposure: Jumping Universes

Let's continue in the `my_cool_project` folder we made on your Desktop in the last lesson.

1. Open Terminal: `cd Desktop/my_cool_project`
2. Check your current branch: `git branch` (It will show `main` or `master` with a * next to it).
3. Create a new branch: `git branch testing-stuff`
4. Jump into it: `git checkout testing-stuff`
5. Create a brand new file: `touch test.txt`
6. `git add .` then `git commit -m "Added a test file in the parallel universe"`
7. Look at your folder (type `ls` or just look at it on your Desktop). You see `test.txt`!
8. **THE MAGIC JUMP:** switch back to the main timeline: `git checkout main`
9. Look at your folder again (`ls`). **THE `test.txt` FILE DISAPPEARED!** 
10. Jump back: `git checkout testing-stuff`. The file is back!
