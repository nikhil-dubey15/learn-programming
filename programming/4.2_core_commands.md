# 4.2 Core Git Commands: Saving the Game

In the last lesson, we learned that Git is a "Save Game" system. Now let's learn the exact Terminal commands to save our code.

**The Analogy: Taking a Photograph**
Saving a file in Git is a three-step process. Imagine you are taking a family portrait.
1. **The Room:** Everyone is running around the living room (Working Directory).
2. **The Stage:** You tell everyone to stand still on the rug and smile (The Staging Area).
3. **The Flash:** You press the button and take the picture, freezing that exact moment in time permanently (The Commit).

## The 5 Essential Commands

### 1. `git init` (Start the Game)
* **What it means:** "Turn this regular, boring folder into a magical Git tracked folder."
* **When to use it:** *Only once*, at the very beginning of a brand new project.

### 2. `git clone` (Download the Game)
* **What it means:** "Download a project that already exists on GitHub onto my laptop."
* **When to use it:** When you join a new company and need to download their code to start working.

### 3. `git status` (Looking around the Room)
* **What it means:** "Git, what has changed since my last save?"
* **When to use it:** ALL THE TIME. You will type this command 50 times a day. It tells you exactly which files you modified, created, or deleted.

### 4. `git add` (The Stage)
* **What it means:** "Hey Git, look at this specific file. I want to include it in my next photograph."
* **How to use it:** 
  - `git add index.html` (Put just this one file on the stage).
  - `git add .` (The dot means "Put EVERY file I changed on the stage right now!").

### 5. `git commit` (The Flash / The Save Game)
* **What it means:** "Take the photograph! Permanently save the state of all the files currently on the stage."
* **How to use it:** `git commit -m "Added a cool new login button"`
* *(The `-m` stands for message. You MUST leave a short note describing what you changed so future developers know what this save game contains).*

---

## 🛠️ Practical Exposure: Let's make a save!

1. Open your Terminal and go to your Desktop: `cd Desktop`
2. Make a new project folder: `mkdir my_cool_project`
3. Go inside it: `cd my_cool_project`
4. **Turn on Git:** `git init` (You will see a message saying "Initialized empty Git repository").
5. Check the status: `git status` (It will say nothing to commit).
6. Create a file: `touch script.js`
7. Check the status: `git status` (Notice `script.js` is now listed in red! Git sees it, but it's not saved yet).
8. Move it to the stage: `git add .`
9. Check status again: `git status` (Notice `script.js` turned green! It is on the stage, smiling for the camera).
10. Take the photo: `git commit -m "Created my first file"`

You just successfully created a version control checkpoint!
