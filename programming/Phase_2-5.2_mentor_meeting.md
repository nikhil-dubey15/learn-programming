# Phase 2 - 5.2 Meeting with Mentors: The Code Review

You have written your very first script (`app.js`). Now, it's time to show it to the experts.

In the software industry, you will *never* deploy code to a live website without having a Senior Developer read it first. This process is called a **Code Review**.

## 1. Showing Your Script
During your meeting today, open up VS Code and show them your `app.js` file. Run the `node app.js` command in front of them so they can see the magic happen!

### Ask the Golden Question:
> *"Hey team, I wrote this script using variables, loops, and async/await. Can you give me a quick Code Review? How would a Senior Developer write this exact same code differently?"*

By asking this, you are forcing them to teach you best practices. They might teach you about modern `map` or `forEach` loops instead of the old-school `for` loop you used!

## 2. Talk about Database Architecture
Today you learned about SQL, NoSQL, and Caching with Redis. These are massive decisions made by engineering teams. Ask your mentors about the decisions made at their company.

### Questions for them:
* *"In your main application, do you use a Relational (SQL) database or a NoSQL database? Why did the company choose that one?"*
* *"Where do you guys use Caching? Have you ever had a scary bug caused by 'stale data' in a cache?"*

## 3. Discussing the Next Steps (Checking in on Phase 3)
Tomorrow is Day 3. Tomorrow you will combine Day 1 (Browsers & Git) and Day 2 (JS Programming) to build **Automated QA Tests**.

### Question for them:
* *"I am moving into QA Automation Testing tomorrow. At this company, which automation framework do you use? (e.g., Cypress, Playwright, Selenium, RestAssured)?"* 
* *"If I am going to write my very first UI Automated test tomorrow, which framework do you recommend I learn first?"*
