# 4.5 Merge Conflicts: When Universes Collide!

Every developer—even the ones with 20 years of experience—fears the dreaded **Merge Conflict**. But don't panic! It is a completely normal part of working on a team.

## How does a Conflict happen?
Imagine a document with a character biography:
* Line 10: `Character Hair Color: Brown`

1. **Developer A** logs in, changes Line 10 to `Character Hair Color: Blonde`, commits, and pushes it to GitHub.
2. **Developer B** (you) didn't `git pull` this morning. You are looking at the old file. You change Line 10 to `Character Hair Color: Red`, commit, and try to push to GitHub.
3. **GitHub stops you!** It throws a giant error! 

GitHub says: *"Uh oh! Developer A said the hair is Blonde. You are saying the hair is Red. You both edited the exact same line at the exact same time. I am just a robot, I don't know who is right! You humans need to fix this."*

**This is a Merge Conflict.**

## How to resolve it in your Code Editor

When a conflict happens, Git will literally inject weird markings into your code file so you know where the fight is happening. If you open the file in your editor (like VS Code), it will look like this:

```text
<<<<<<< HEAD (Your changes)
Character Hair Color: Red
=======
Character Hair Color: Blonde
>>>>>>> origin/main (Developer A changes)
```

### The Fix (3 Steps):
1. **Talk to the human!** Ask Developer A: *"Hey, we both touched line 10. Did the client want red or blonde hair?"*
2. **Delete the garbage.** Manually delete the `<<<<<<<`, `=======`, and `>>>>>>>` markers.
3. **Keep the winner.** Delete the wrong code and keep the right code. Like this:
   ```text
   Character Hair Color: Blonde
   ```
4. Save the file. Run `git add .` and `git commit -m "Resolved merge conflict on hair color"`.
5. You did it!

## Why VS Code is amazing for this
Modern code editors like VS Code are very smart. When you get a merge conflict, you don't actually have to delete the markers manually. 

VS Code will highlight the code block in bright colors and show you three buttons you can just click:
* **[Accept Current Change]** (Keep your code, throw away theirs)
* **[Accept Incoming Change]** (Keep their code, throw away yours)
* **[Accept Both Changes]** (Keep both on two separate lines)
