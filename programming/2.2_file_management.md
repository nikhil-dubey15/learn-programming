# 2.2 File Management: Creating and Moving Boxes

Now that you know how to walk around the "house" (your computer) using the Terminal, let's learn how to create things, move them around, and throw them away.

**The Analogy: Playing with Magic LEGOs**
Think of files and folders as LEGO blocks. With a single text command, you can make a new block appear, copy it, move it to another room, or vanish it forever!

## The Core Commands

### 1. `mkdir` (Make Directory)
* **What it means in English:** "Build a new empty box (folder) right here."
* **How to use it:** `mkdir my_new_folder`
* **Practical Check:** If you run this, you just created a folder named `my_new_folder` in your current room.

### 2. `touch`
* **What it means in English:** "Create a new, empty piece of paper (file)."
* **How to use it:** `touch notes.txt`
* **Practical Check:** You just created a blank text file named `notes.txt`. 

### 3. `cp` (Copy)
* **What it means in English:** "Make an exact duplicate of this."
* **How to use it for a file:** `cp notes.txt backup.txt` (Makes a copy of `notes.txt` and calls it `backup.txt`).
* **How to use it for a folder:** `cp -r my_new_folder copied_folder` (The `-r` means "recursive", which is computer-speak for "copy the box AND everything inside it").

### 4. `mv` (Move / Rename)
* **What it means in English:** "Move this thing somewhere else OR change its name."
* **How to use it to move:** `mv notes.txt my_new_folder/` (Moves the file into the folder).
* **How to use it to rename:** `mv backup.txt super_secret_notes.txt` (Just changes the name of the file).

### 5. `rm` (Remove)
* **⚠️ WARNING: This is dangerous!** When you use `rm`, it doesn't go to your Trash bin. It gets deleted forever immediately!
* **How to use it for a file:** `rm secret_notes.txt`
* **How to use it for a folder:** `rm -r my_new_folder` (Again, `-r` tells it to destroy the whole box and everything inside).

## 🛠️ Practical Exposure: The Magic Builder

Let's build a tiny project completely using text!

1. Open your Terminal. Use `cd` to go somewhere safe, like your Desktop: `cd Desktop`
2. Create a new folder: `mkdir QA_Testing_Lab`
3. Go inside the new folder: `cd QA_Testing_Lab`
4. Create a new empty file: `touch test_case_1.txt`
5. Verify it's there: `ls`
6. Make a backup copy: `cp test_case_1.txt test_case_backup.txt`
7. Rename the backup: `mv test_case_backup.txt test_case_2.txt`
8. Verify again: `ls` (You should see both files!)
9. Go back one level: `cd ..`
10. Finally, clean up after yourself by deleting the folder: `rm -r QA_Testing_Lab`
