# 4.1 Git Basics: The Time Machine for Code

Welcome to Version Control! As a QA and Developer, you will write code, delete code, and accidentally break code. We need a way to save our progress so we can always "undo" mistakes.

**The Analogy: Playing a Video Game**
Imagine playing a really hard video game (like Mario). Before you try to jump over a huge pit of lava, what do you do? You hit the **"Save Game"** button.

If you jump and fall into the lava, you don't have to start the whole game over from Level 1! You just load your last "Save Game" and try the jump again.

**Git is simply a "Save Game" system for your code.**

## What is Git?
Git is a free tool you install on your computer. It watches a folder (like your project folder) and keeps track of every single change you make to every single file.

## Why do we use it?
1. **Time Travel:** If you delete a file by accident, Git can bring it back instantly.
2. **Experimentation:** You can try to build a crazy new feature. If it fails, you can tell Git, "Erase all that and put my code back exactly how it was yesterday."
3. **Collaboration:** It allows 10 developers to work on the exact same file at the exact same time without overwriting each other's work.

## Git vs. GitHub (The Biggest Confusion)
Many beginners think these are the same thing. They are NOT!

* **Git:** The actual tool running locally on your laptop. (Like Microsoft Word).
* **GitHub (or GitLab / BitBucket):** A website where you upload and store your Git "Save Games" so other people can download them. (Like Google Drive or Dropbox, but built specifically for code).
