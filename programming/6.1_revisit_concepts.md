## The "Explain it to a 5-Year-Old" Test
The best way to know if you truly understand a technical concept is if you can explain it in simple, non-technical words using an analogy. 

**Try to answer these questions out loud right now. If you get stuck, go back and read the markdown file again!**

### 1. Computer Architecture (Files 1.1 - 1.2)
* Explain the difference between the CPU and the RAM using the "Kitchen" analogy. 
* If my computer has 16GB of RAM, but I open 20GB worth of Chrome tabs, why does my computer suddenly become super slow? What is the OS doing?

### 2. The Terminal (Files 2.1 - 2.4)
* What is the difference between `pwd` and `ls`?
* I am currently standing in the `/Users/nikhildubey` folder. I want to look inside a file called `notes.txt` located on my Desktop. What is the **Relative Path** command to read that file? What is the **Absolute Path** command?

### 3. Networking (Files 3.1 - 3.4)
* Explain the difference between an IP Address and a Port using an Apartment Building analogy.
* What exactly happens when I type `amazon.com` into my browser? How does the DNS act like my phone's Contact App?
* If a webpage loads perfectly but a single image is missing, is the browser probably using TCP or UDP? Why?
* If I click a broken link and see a `404 Not Found` error, who made the mistake: Me (the user) or the Server?

### 4. Git & GitHub (Files 4.1 - 4.5)
* Why do we create Branches instead of doing all our work on the `main` timeline?
* Explain what a Pull Request (PR) is. 
* You and a coworker both try to change Line 10 of a file at the exact same time. What happens, and how do you fix it?

### 5. DevTools (Files 5.1 - 5.4)
* I click a "Submit" button on a website, but nothing happens. Which DevTools tab should I open first to find the red error message?
* A website remembers that my favorite background color is Dark Mode, even though I closed my laptop yesterday. Is that saved in a Cookie, Session Storage, or Local Storage?
